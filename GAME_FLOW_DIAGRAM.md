## Critical Flow Points

### 1. Race Start Trigger
- **Where**: `consumers.py` - `connect()` method
- **Condition**: Both player1 and player2 connected
- **Action**: Set start_time, status='in_progress', broadcast race_started

### 2. Timer Synchronization
- **Where**: `game_board.js` - `startTimers()` function
- **Mechanism**: Parse server start_time, calculate elapsed, update every second
- **Retry**: Exponential backoff if DOM not ready

### 3. Move Validation
- **Client**: `validateMove()` - immediate visual feedback
- **Server**: `handle_move()` - authoritative validation
- **Sync**: lastInputCell tracking for error clearing

### 4. Auto-Submit
- **Trigger**: `isLocalBoardComplete() && isLocalBoardValid() && !gameFinished`
- **Guard**: gameFinished flag prevents duplicates
- **Timing**: Checked after every move, debounced via setTimeout

### 5. Result Finalization
- **Concurrency**: `transaction.atomic()` + `select_for_update()`
- **Idempotent**: Check existing result first
- **Fallback**: IntegrityError catch for race conditions

### 6. Modal Display
- **Bootstrap**: Requires bootstrap.Modal initialization
- **Content**: Dynamic HTML injection based on winner/loser
- **Timing**: Waits for race_finished message from server
